1. GET /getword - случайное слово из БД, ответ 200

2. GET /word/{name} - выдает слово с указанным name
если не найдено в БД - ответ 404
если успешно - ответ(200)

3. GET /word/{id} - выдает слово с указанным id
если не найдено в БД - ответ 404
если успешно - ответ(200)

4. POST /word - попытка добавить новую запись в БД
если упешно - ответ 201
если же уже есть word с таким же name, то ответ 409. То есть обнолвение записи не производится
Пример request body {"name": "window","translate": "окно"}
Пример request body {"id": 2,"name": "window","translate": "окно"} - id в body(!) игнорируется

5. PUT /word/{id} - попытка положить новое слово по id.
если id <= 0 - ответ 400
если записи с таки id не найдено - ответ 404
если новое name уже есть в другой записи - ответ 409
если успешно - ответ 200
Пример request body {"name": "window","translate": "окно"}
Пример request body {"id": 2,"name": "window","translate": "окно"} - id в body(!) игнорируется. Играет роль именно id в url

6. PUT /word - попытка добавить/отредактировать запись в БД.
если записи с таким name нет, будет добавлена новая, ответ 200 (!) (не 201,временный баг)
если запись с таким name есть, будет отредактирована текущая (отредактиовано поле translate), ответ 200
Пример request body {"name": "window","translate": "окно"}
Пример request body {"id": 2,"name": "window","translate": "окно"} - id в body(!) игнорируется

7. DELETE /word/{id} - уделание слова с указанным id
если записи с таким id не найдено - ответ 404
если успешно - ответ 204

8. DELETE /word/{name} - уделание слова с указанным name
если записи с таким name не найдено - ответ 404
если успешно - ответ 204
